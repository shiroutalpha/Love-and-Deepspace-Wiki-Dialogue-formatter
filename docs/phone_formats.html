Phone Block

Use @type - name ... @@ to create phone interactions (text messages, voice/video calls).

  • @voice - Name – voice call
  • @video - Name – video call
  • @text message - Name – text message
  • @@ – end of phone block
  • Character is optional (can omit - Character)

Basic phone block

@voice - Rafayel
Rafayel: Hello, can you hear me?
MC: Yes, I can hear you clearly.
Rafayel: Good, let's continue.
@@
Result ▼
{{Script/phone|Voice|Rafayel| Dialogue =
{{Script/dialogue|Rafayel|Hello, can you hear me?}}
{{Script/dialogue|MC|Yes, I can hear you clearly.}}
{{Script/dialogue|Rafayel|Good, let's continue.}}
}}
@text message - Zayne
Zayne: I need you to come to the lab.
MC: On my way.
@@
Result ▼
{{Script/phone|Text Message|Zayne| Dialogue =
{{Script/dialogue|Zayne|I need you to come to the lab.}}
{{Script/dialogue|MC|On my way.}}
}}

Phone block without character name

@video
Rafayel: Can you see me?
MC: Yes, I can see you.
@@
Result ▼
{{Script/phone|Video| Dialogue =
{{Script/dialogue|Rafayel|Can you see me?}}
{{Script/dialogue|MC|Yes, I can see you.}}
}}

Usage

  • If the prefix or suffix is invalid → the entire line is marked as invalid (highlighted red).
Xavier: <Galaxy Kid: Wave>
Result ▼
{{Script/animated emoji|Xavier|Galaxy Kid: Wave}}
Rafayel: <Artsy Birb: Bye>
Result ▼
{{Script/animated emoji|Rafayel|Artsy Birb: Bye}}
Note:
  • Animated emoji lines must contain only the emoji tag – no other text. If there is any other text (e.g., Rafayel: Hello <Artsy Birb: Bye>), the line is considered invalid.
  • The prefix and suffix are validated against predefined lists. If the combination is invalid, the entire line will be marked as invalid.
