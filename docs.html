Overview

This tool converts plain text input into wikitext format compatible with loveanddeepspace.wiki.gg.

It supports multiple input formats including:

  • Dialogue lines with optional expression and character override
  • Type tags (Radio, Moment, Text Message)
  • Script block with configurable parameters

The conversion happens automatically as you type, and errors are highlighted in red with a status summary.

Dialogue Formats

1. Basic dialogue

Character: Dialogue

Rafayel: Hello
Result ▼
{{Script/dialogue|Rafayel|Hello}}

2. Dialogue with expression

Character (expression): Dialogue

Rafayel (smiling): Hello
Result ▼
{{Script/dialogue|Rafayel|Hello|Expression=smiling}}

3. Dialogue with character override

Character - override: Dialogue

Rafayel - ???: Hello
Result ▼
{{Script/dialogue|Rafayel|Hello|Character Override=???}}

Note: The override is useful when you want to display a different character name in the script.

Type Tags

Use these tags to insert special script elements.

  • <type/Radio> → {{Script/type|Radio}}
  • <type/Moment> → {{Script/type|Moment}}
  • <type/Text Message> → {{Script/type|Text Message}}

Case‑insensitive: <type/radio>, <type/moment>, <type/text message> also work.

<type/Radio>
Result ▼
{{Script/type|Radio}}

Script Block

Wrap your script with <Script> to add {{Script/start}} and {{Script/end}}.

You can also include parameters inside the block. Only the following keys are allowed:

  • Toggle — must be No Toggle (for false) or Collapsed (for Collapsed)
  • Scheme — must be Dark or Light
  • Theme
  • Title
  • Hide Title
  • Previous Chapters
  • Next Chapters
  • Edit Script

Simple <Script>

<Script>
Rafayel: Hello
Result ▼
{{Script/start}}
{{Script/dialogue|Rafayel|Hello}}
{{Script/end}}

With parameters

<Script
Title: Birthday 2026
Toggle: No Toggle
Scheme: Light
Theme: Birthday>
Rafayel: Hello
Result ▼
{{Script/start|Title=Birthday 2026|Toggle=false|Scheme=Light|Theme=Birthday}}
{{Script/dialogue|Rafayel|Hello}}
{{Script/end}}
Important: The Toggle value is strictly validated. Only No Toggle (becomes false) and Collapsed (becomes Collapsed) are accepted. Any other value will cause the entire script block to be marked as invalid and highlighted in red.

Validation & Error Highlighting

Lines that do not match any recognised format will be highlighted in red in the input backdrop.

  • Unrecognised dialogue lines (e.g. missing colon) → red
  • Invalid <Script> block (e.g. wrong Toggle value) and Invalid keys inside → all lines inside the block turn red

A status message at the bottom shows how many lines were converted and how many were skipped (with a preview).
