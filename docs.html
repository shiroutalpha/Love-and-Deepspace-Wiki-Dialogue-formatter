Overview

This tool converts plain text input into wikitext format compatible with loveanddeepspace.wiki.gg.

It supports multiple input formats including:

  • Dialogue lines with optional expression and character override
  • Type tags (Radio, Moment, Text Message, Reset)
  • Script block with configurable parameters
  • Heading, Break, Context, Narration, Quote, Interaction, Choice, and Phone elements

The conversion happens automatically as you type, and errors are highlighted in red with a status summary.

Dialogue Formats

1. Basic dialogue

Character: Dialogue

Rafayel: Hello
Result ▼
{{Script/dialogue|Rafayel|Hello}}

2. Dialogue with expression

Character (expression): Dialogue

Rafayel (smiling): Hello
Result ▼
{{Script/dialogue|Rafayel|Hello|Expression=smiling}}

3. Dialogue with character override

Character - override: Dialogue

Rafayel - ???: Hello
Result ▼
{{Script/dialogue|Rafayel|Hello|Character Override=???}}

Note: The override is useful when you want to display a different character name in the script.

Script Block

Wrap your script with <Script> to add {{Script/start}} and {{Script/end}}.

You can also include parameters inside the block. Only the following keys are allowed:

  • Title
  • Hide Title — must be Yes (for true) or No (for false)
  • Toggle — must be Yes (for true) or No (for false) or Collapsed (for Collapsed)
  • Scheme — must be Dark or Light
  • Theme
  • Previous Chapters — must be Link 1,, Display Name 1;; Link 2,, Display Name 2
  • Next Chapters — must be Link 1,, Display Name 1;; Link 2,, Display Name 2
  • Edit Script

Simple <Script>

<Script>
Rafayel: Hello
Result ▼
{{Script/start}}
{{Script/dialogue|Rafayel|Hello}}
{{Script/end}}

With parameters

<Script
Title: Birthday 2026
Toggle: No Toggle
Scheme: Light
Theme: Birthday>
Rafayel: Hello
Result ▼
{{Script/start|Title=Birthday 2026|Toggle=false|Scheme=Light|Theme=Birthday}}
{{Script/dialogue|Rafayel|Hello}}
{{Script/end}}
Important: The Toggle value is strictly validated. Only No Toggle (becomes false) and Collapsed (becomes Collapsed) are accepted. Any other value will cause the entire script block to be marked as invalid and highlighted in red.

Type Tags

Use these tags to insert special script elements.

  • <type/Radio> → {{Script/type|Radio}}
  • <type/Moment> → {{Script/type|Moment}}
  • <type/Text Message> → {{Script/type|Text Message}}
  • <type/Reset> → {{Script/type|Reset}}

Case‑insensitive: <type/radio>, <type/moment>, <type/text message>, <type/reset> also work.

<type/Radio>
Result ▼
{{Script/type|Radio}}
<type/Reset>
Result ▼
{{Script/type|Reset}}

Heading

Use ## to create a heading. You can optionally add a subtitle inside < >.

  • ## Text → {{Script/heading|Text}}
  • ## Text <Subtitle> → {{Script/heading|Text|Subtitle=Subtitle}}
## Introduction
Result ▼
{{Script/heading|Introduction}}
## Scene 1 <The Beginning>
Result ▼
{{Script/heading|Scene 1|Subtitle=The Beginning}}

Note: To add custom alignment, you can manually edit the output to add |Align Text=left, |Align Text=center, or |Align Text=right.

Break

Use three dashes --- on a line by itself to insert a script break.

---
Result ▼
{{Script/break}}

Context

Use <Context> to insert a context block. You can optionally specify a type (e.g., Title, Instructions).

  • <Context> Text → {{Script/context|Text}}
  • <Context Type> Text → {{Script/context|Type=Type|Text}}
<Context> This is a context note.
Result ▼
{{Script/context|This is a context note.}}
<Context Title> Important information
Result ▼
{{Script/context|Type=Title|Important information}}
<Context Instructions> Follow these steps
Result ▼
{{Script/context|Type=Instructions|Follow these steps}}

Narration

Use > for standard narration or >> for large narration blocks (supports multiple lines).

Single-line narration

> Text → {{Script/narration|Text}}

> The room was silent.
Result ▼
{{Script/narration|The room was silent.}}

Multi-line large narration

>> Text1 followed by >> Text2 on subsequent lines will be combined into one block.

>> The wind howled.
>> The storm approached.
Result ▼
{{Script/narration large|The wind howled.;;The storm approached.}}

The converter automatically joins multiple >> lines with ;;. The block closes when a different format (dialogue, heading, etc.) is encountered.

Quote

Use ^ to insert a quote, optionally with a source.

  • ^Text → {{Script/quote|Text}}
  • ^Text <Sources> → {{Script/quote|Text|Sources}}
^To be, or not to be.
Result ▼
{{Script/quote|To be, or not to be.}}
^That's one small step for man. <Neil Armstrong>
Result ▼
{{Script/quote|That's one small step for man.|Sources=Neil Armstrong}}

Interaction

Use !! to create interactive elements. Format: !! Type - Text <additional>.

  • Type – one word (e.g., Click, Tap, Swipe, Drag, Arrow)
  • Text – description, can have spaces (optional)
  • additional (optional) – inside < >:
    • flip → Flip=true
    • 45 (1–360) → Rotate=45
    • 45//flip → Rotate=45|Flip=true
!! Click - Tap to continue
Result ▼
{{Script/interaction|Click|Tap to continue}}
!! Swipe - Swipe left <90>
Result ▼
{{Script/interaction|Swipe|Swipe left|Rotate=90}}
!! Drag - Drag object <flip>
Result ▼
{{Script/interaction|Drag|Drag object|Flip=true}}
!! Tap - Tap here <45//flip>
Result ▼
{{Script/interaction|Tap|Tap here|Rotate=45|Flip=true}}
!! Arrow
Result ▼
{{Script/interaction|Arrow}}
!! Arrow <180>
Result ▼
{{Script/interaction|Arrow|Rotate=180}}
!! Arrow <flip>
Result ▼
{{Script/interaction|Arrow|Flip=true}}
!! Arrow <90//flip>
Result ▼
{{Script/interaction|Arrow|Rotate=90|Flip=true}}

Choice

Use ??? ... ??? to create a tabbed choice menu. Format:

  • ??? – start of choice block
  • ? Title 1 – tab title, followed by content
  • ? Title 2 – next tab title, followed by content
  • ??? – end of choice block

The content inside each tab is automatically converted (dialogue, type tags, headings, etc.).

???
? Help Rafayel
Rafayel: Thank you for helping me!
Zayne: That was a wise choice.
? Ignore Rafayel
Rafayel: Why didn't you help me?
Sylus: That was a mistake.
???
Result ▼
<tabber>
|-|Help Rafayel =
{{Script/dialogue|Rafayel|Thank you for helping me!}}
{{Script/dialogue|Zayne|That was a wise choice.}}
|-|Ignore Rafayel =
{{Script/dialogue|Rafayel|Why didn't you help me?}}
{{Script/dialogue|Sylus|That was a mistake.}}
</tabber>

Phone

Use @type - name ... @@ to create phone interactions (text messages, voice/video calls).

  • @voice - Name – voice call
  • @video - Name – video call
  • @text message - Name – text message
  • @@ – end of phone block
  • Name is optional (can omit - Name)

The content inside is automatically converted (dialogue, type tags, headings, etc.).

@voice - Rafayel
Rafayel: Hello, can you hear me?
MC: Yes, I can hear you clearly.
Rafayel: Good, let's continue.
@@
Result ▼
{{Script/phone|Voice|Rafayel| Dialogue =
{{Script/dialogue|Rafayel|Hello, can you hear me?}}
{{Script/dialogue|MC|Yes, I can hear you clearly.}}
{{Script/dialogue|Rafayel|Good, let's continue.}}
}}
@text message - Zayne
Zayne: I need you to come to the lab.
MC: On my way.
@@
Result ▼
{{Script/phone|Text Message|Zayne| Dialogue =
{{Script/dialogue|Zayne|I need you to come to the lab.}}
{{Script/dialogue|MC|On my way.}}
}}
@video
Rafayel: Can you see me?
MC: Yes, I can see you.
@@
Result ▼
{{Script/phone|Video| Dialogue =
{{Script/dialogue|Rafayel|Can you see me?}}
{{Script/dialogue|MC|Yes, I can see you.}}
}}

Validation & Error Highlighting

Lines that do not match any recognised format will be highlighted in red in the input backdrop.

  • Unrecognised dialogue lines (e.g. missing colon) → red
  • Invalid <Script> block (e.g. wrong Toggle value) and Invalid keys inside → all lines inside the block turn red
  • Invalid Interaction options (e.g. wrong rotate value or invalid flip) → line turns red
  • Invalid Phone type → line turns red

A status message at the bottom shows how many lines were converted and how many were skipped (with a preview).
